# Research Gallery Architect

**[Open the tool → researchgallery.html](researchgallery.html)**

A "low-code" digital humanities utility designed to bridge the gap between archival research and digital presentation. This tool allows scholars and curators to transform raw media files into structured, annotated, and sortable HTML galleries.

## 🎓 Intended Use Cases

### 1. The PhD Research Workbench
Perfect for doctoral candidates managing diverse primary sources. Use this tool to:
* Organize "bits" of research into a chronological or thematic deck.
* Write critical annotations alongside visual evidence.
* Export a clean, semantic HTML appendix for digital dissertations or personal portfolios.

### 2. The Museum "Curated Highlights" Exhibit
Designed for curators who want to showcase high-value items (including 3D GIFs or archival scans) without the technical overhead of a full database. 
* Optimized for curated exhibits (10–100 items).
* Provides professional "Plinth" (Gallery) and "Workbench" (Technical) styling.

## 🚀 How to Use the Tool

1. **Launch:** Open `researchgallery.html` in any modern browser (Microsoft Edge, Chrome, etc.).
2. **Import:** Drag and drop your images or GIFs into the "Visual Evidence" zone.
3. **Annotate:** Fill in the metadata fields (Title, Author, Date, and Analysis).
4. **Organize:** Use the **Sort** buttons to view your collection by Date or Title.
5. **Export:** * **Copy Code:** Grabs the HTML snippet for use in existing websites/blogs.
   * **Download HTML:** Generates a standalone gallery file.

## ⚠️ Important Implementation Note
To ensure your images appear in the exported gallery:
* Save your images in a local folder on your computer.
* Place your **Downloaded HTML file** in that same folder. 
* The tool uses the filenames (e.g., `map_1920.jpg`) to link the data; keeping the file and images together maintains the "connection."
